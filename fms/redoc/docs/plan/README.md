# Planning Artifacts

The planning folder codifies the delivery cadence and guardrails for the FMS program.

## Documents
- `roadmap.md` — Phase-by-phase breakdown from Foundations through Hardening, with milestone checklists tied directly to upstream specs. Each phase links to the relevant docs (UI, integrations, technical plans) so work can be tracked end-to-end.
- `spec-freeze-v1.0.md` — Formal freeze notice dated 2025-11-11 that locks MVP scope, enumerates delivered artifacts, records key decisions (datastores, dedupe strategy, retention, billing defaults), and lists post-freeze backlog items.

## How to Use
- Reference the roadmap during sprint planning to ensure work lines up with the correct phase and acceptance criteria.
- Consult the spec freeze before accepting new scope; any additions must be logged in `docs/work/todo.md` with impact analysis.

These documents prevent scope drift and keep all teams aligned on launch commitments.
