# Requirements Index

`warehouse-delivery-linked.md` traces every section of the source requirements document (`/warehouse-delivery.md`) to the design artifacts that implement it. It also captures freeze-era checklists that gate launch readiness.

## Sections
- **2.x Functional Requirements** — Links to integrations, scheduling, routing, tracking, feedback loops, dashboard widgets, and accessibility commitments with explicit checklists per subsection.
- **3. Non-Functional Requirements** — Ties performance, availability, privacy, security, and accessibility demands to technical specs, DR plans, and settings.
- **Appendices** — References industry addenda, OpenAPI specs, and the RTM so auditors see the entire documentation chain.

Use this folder as the jumping-off point when validating scope (spec freeze) or preparing change requests—the linked checklist makes it obvious where a new or modified requirement must land in the documentation tree.
