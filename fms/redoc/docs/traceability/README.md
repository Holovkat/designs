# Traceability Artifacts

Traceability links business requirements to concrete specifications, schemas, and test plans.

## Files
- `requirements-matrix.md` — Phase-by-phase matrix showing how each requirement from `warehouse-delivery.md` maps to Convex schemas, integrations, APIs, UI specs, workflows, configuration, admin tools, and reports. Includes acceptance-criteria breakdowns for dashboard, routing, driver app, and inventory reconciliation requirements.
- `rtm.md` — Compact Requirements Traceability Matrix summarizing requirement IDs, the authoritative spec, and the implementation artifact for quick audits.

## Usage
- Use the matrix to spot gaps or verify that a requirement has an implementation path across stack layers.
- Reference the RTM when preparing compliance or release checklists; it serves as the high-level audit log of requirement coverage.

Both documents stay synchronized with the roadmap and spec freeze, ensuring scope remains locked while still traceable end-to-end.
