# Verify OKF Deployment

Phase 8 of the OKF deployment workflow. Run these verification steps to confirm the deployment is complete and correct.

## 1. Verify Concept Counts

Count concepts per directory and verify `index.md` counts match:

```bash
for dir in architecture components domain decisions process deprecation state; do
  count=$(find knowledge/$dir -name "*.md" -not -name "index.md" | wc -l)
  echo "  $dir: $count"
done
```

Compare the output to the counts in `knowledge/index.md`. Every mismatch must be fixed by updating the index. See [Index Structure](../domain/index-structure.md).

## 2. Verify No AFFiNE References

```bash
grep -i affine AGENTS.md
```

This should return no results. If AFFiNE references remain, complete the [AGENTS.md migration](./migrate-agents-md.md).

## 3. Verify No docs/agents/ References

```bash
grep "docs/agents/" AGENTS.md
```

This should return no results. If `docs/agents/` references remain, update them to point to `knowledge/process/` equivalents.

## 4. Verify OKF Knowledge Bundle Section

```bash
grep "OKF Knowledge Bundle" AGENTS.md
```

This should return a match. The section should contain agent onboarding steps, after-completing-work instructions, curation description, concept types table, and rules.

## 5. Verify Post-Commit Hook

```bash
git config core.hooksPath
ls -la .githooks/post-commit
```

- `git config core.hooksPath` should return `.githooks`.
- `.githooks/post-commit` should exist and be executable (permission bits include `x`).

See [Hook System](../architecture/hook-system.md) and [Installer Design](../architecture/installer-design.md).

## 6. Verify Viewer Files

```bash
ls -lh knowledge/viz.html
ls -lh knowledge/viewer.html
ls -lh knowledge/generate-viz.js
```

- `viewer.html` should exist (copied by installer).
- `generate-viz.js` should exist (copied by installer).
- `viz.html` should exist (generated by running `generate-viz.js`). File size should be 100KB-500KB depending on concept count.

Open `viz.html` in a browser and verify:
- Browse tab shows folder tree with all concept directories.
- Graph tab shows the knowledge graph with typed nodes and edges.
- Clicking a concept shows its rendered markdown.
- Mermaid diagrams render inline (if any concepts have mermaid code blocks).
- Relative links in concept bodies navigate within the viewer.
- Search filtering works.

See [Viewer Architecture](../architecture/viewer-architecture.md) and [Viz Generator Design](../architecture/viz-generator-design.md).

## 7. Verify Knowledge Structure

```bash
ls knowledge/
```

Should show: `index.md`, `log.md`, `inbox/`, `architecture/`, `components/`, `domain/`, `decisions/`, `process/`, `deprecation/`, `state/`, `viewer.html`, `generate-viz.js`, `viz.html`.

Each subdirectory should have an `index.md` file.

## Post-Verification

If all checks pass, the deployment is complete. The project is ready for ongoing OKF usage as described in the [deployment workflow](./deploy-okf.md).

If any checks fail, return to the relevant phase and fix the issue before considering the deployment complete.
